# LinkHub 环境部署

本文记录当前 Windows 开发机的可用环境，后续换机或重建环境时按这里恢复。

## 已安装组件

- Rust/Cargo: 通过 rustup 安装。
- JDK 17: `C:\Program Files\Eclipse Adoptium\jdk-17.0.19.10-hotspot`
- Node.js LTS: 用于桌面端前端工具链。
- Visual Studio Build Tools C++: 用于 Rust/native 构建依赖。
- Android Studio: 用户已安装。
- Android SDK: `C:\Users\<用户名>\AppData\Local\Android\Sdk`
- Android command line tools: `cmdline-tools\latest`
- Android SDK packages:
  - `platforms;android-34`
  - `build-tools;34.0.0`
  - `system-images;android-34;google_apis;x86_64`
  - `ndk;28.2.13676358`（交叉编译 `liblinkhub_core.so`）
- Android AVD: `LinkHub_API_34`
- Rust 安卓 target: `aarch64-linux-android`、`x86_64-linux-android`（`rustup target add`）
- `cargo-ndk`（`cargo install cargo-ndk`，当前 4.1.2）

## 环境变量

用户级环境变量已设置：

```powershell
ANDROID_HOME=C:\Users\<用户名>\AppData\Local\Android\Sdk
ANDROID_SDK_ROOT=C:\Users\<用户名>\AppData\Local\Android\Sdk
JAVA_HOME=C:\Program Files\Eclipse Adoptium\jdk-17.0.19.10-hotspot
```

用户 `Path` 应包含：

```text
%ANDROID_HOME%\platform-tools
%ANDROID_HOME%\emulator
%ANDROID_HOME%\cmdline-tools\latest\bin
%JAVA_HOME%\bin
```

如果当前 PowerShell 会话没有刷新环境，可临时执行：

```powershell
$sdk='C:\Users\<用户名>\AppData\Local\Android\Sdk'
$jdk='C:\Program Files\Eclipse Adoptium\jdk-17.0.19.10-hotspot'
$env:ANDROID_HOME=$sdk
$env:ANDROID_SDK_ROOT=$sdk
$env:JAVA_HOME=$jdk
$env:Path=[Environment]::GetEnvironmentVariable('Path','Machine')+';'+[Environment]::GetEnvironmentVariable('Path','User')+";$jdk\bin;$sdk\platform-tools;$sdk\emulator;$sdk\cmdline-tools\latest\bin"
```

## 验证命令

Rust core:

```powershell
cd C:\Dev\VSCode\LinkHub\core-rs
cargo test --quiet
```

Desktop Rust:

```powershell
cd C:\Dev\VSCode\LinkHub\desktop\src-tauri
cargo test --quiet
```

Android 构建:

```powershell
cd C:\Dev\VSCode\LinkHub
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\build-android-so.ps1
cd C:\Dev\VSCode\LinkHub\android
.\gradlew.bat :app:assembleDebug
```

Android `.so` 不再随仓库提交，是本地/CI 构建产物。新 clone 或改了 `core-rs` 后，先生成 `android/app/src/main/jniLibs/**/liblinkhub_core.so`，再运行 Gradle：

```powershell
cd C:\Dev\VSCode\LinkHub
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\build-android-so.ps1
```

等价手动命令:

```powershell
$env:ANDROID_NDK_HOME='C:\Users\<用户名>\AppData\Local\Android\Sdk\ndk\28.2.13676358'
cd C:\Dev\VSCode\LinkHub\core-rs
cargo ndk -t arm64-v8a -t x86_64 -o ..\android\app\src\main\jniLibs build --release --lib
```

Android 模拟器烟测:

```powershell
$sdk='C:\Users\<用户名>\AppData\Local\Android\Sdk'
Start-Process -FilePath "$sdk\emulator\emulator.exe" -ArgumentList @('-avd','LinkHub_API_34','-no-window','-no-snapshot','-gpu','swiftshader_indirect','-no-audio') -WindowStyle Hidden
adb wait-for-device
adb install -r C:\Dev\VSCode\LinkHub\android\app\build\outputs\apk\debug\app-debug.apk
adb shell monkey -p com.linkhub.app -c android.intent.category.LAUNCHER 1
adb shell pidof com.linkhub.app
adb emu kill
```

## Android Studio

Android Studio 可以直接打开 `C:\Dev\VSCode\LinkHub\android`。Gradle 使用系统 `JAVA_HOME`，仓库不再提交 `android/local.properties` 或硬编码本机 JDK 路径。
