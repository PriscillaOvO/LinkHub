## 本轮新增（2026-06-17 · 修复 Android 监听服务停止后无法重启 + release 版身份创建失败）

- 背景：真机测试中暴露两个 Android 端问题。
- 问题一 · 监听重启会卡住/重绑失败：`jni_bridge` 的 `startListener` 只把 `LISTENER_RUNNING` 置真后 `spawn` 工作线程，`stopListener` 只把标志置假，**不等待工作线程真正退出**。于是「停止 → 立即启动监听」时，旧线程可能仍持有已绑定的 socket，新线程 `TcpListener::bind` 报「address already in use」；并且旧线程晚退出时会把刚启动的新监听标志清零，导致状态显示错乱。
  - 修复：引入单一收口 `stop_and_join_listener()`——置 `LISTENER_RUNNING=false` 后 `join` 工作线程，确保 socket 已释放；`startListener` 启动前先调用它（处理上一线程尚在拆除的竞态），并把 `JoinHandle` 存入 `LISTENER_HANDLE`；用单调代次 `LISTENER_EPOCH` 做「代际保护」，工作线程退出时只有仍是当前代才清 `LISTENER_RUNNING`，被取代的旧线程晚退出不会再误关新监听；`stopListener` 改为调用同一收口，阻塞到线程退出后再返回。
- 问题二 · release（R8 压缩）版生成身份/读列表偶发失败：R8 静态分析看不到「仅由 Gson 反射填充」的 DTO 字段/构造器，会被裁剪/改名，导致 `gson.fromJson` 返回 null 或抛异常（如生成身份「创建失败」）。
  - 修复：`proguard-rules.pro` 显式 `-keep` 全部 Gson 模型类（`IdentityJson` / `PeerInfoJson` / `PairResultJson` / `TrustedPeer` / `SendResultJson` / `TransmissionHistoryEntry` / `DiscoveredPeerAddress` / `AndroidNetworkHint` / `LinkHubService$ListenerResult`）及其成员；并兜底保留任何标注 `@SerializedName` 的字段。
- 验收：`cargo ndk -t arm64-v8a check --lib`（Android target）通过，仅余既有 `net/protocol.rs` dead_code 警告（新引入的 `LISTENER_EPOCH`/`LISTENER_HANDLE`/`stop_and_join_listener` 已全部被使用，无新增告警）；`core-rs cargo test` 全绿；`desktop/src-tauri cargo check` 通过。已重建两 ABI release `.so` 并 `gradlew :app:assembleRelease` 成功（含 R8 + 资源 shrink + `validateSigningRelease`），产物 `app-release.apk`（约 23.8 MB），`apksigner verify` V2 签名、证书 `CN=LinkHub Dev`（SHA-256 `ae2bbe…a6bc`，与既有验证记录一致，可覆盖安装升级）。**待用户真机复验**：停止→重启监听不再失败、release 版生成身份正常。

## 本轮新增（2026-06-17 · 修复打包版按钮全部失效 / os error 3）

- 现象：安装版里「初始化身份」「开始监听」等按钮点了没反应，切到「设备」页加载状态报 `os error 3`（系统找不到路径）。
- 根因：所有交互按钮用的是行内 `onclick="..."` 事件属性。Tauri 打包构建会向 CSP 的 `script-src` 自动注入 nonce，按 CSP 规范，一旦存在 nonce，`'unsafe-inline'` 会被浏览器忽略，于是**行内事件处理器全部不触发**。标签页切换用的是 `addEventListener`（仍生效），所以切到「设备」页会自动触发 `renderDevicesTab()` → 因尚未初始化身份而报 os error 3，造成「按钮没反应、但加载状态有报错」的现象。
- 修复：在 `app.js` 增加全局事件委托（监听 document 的 click/change，按元素的 `data-act` / `data-change` + `data-a0..a3` 调用对应全局函数）；把六个前端文件里所有行内 `onclick` / `onchange` 改为 `data-act` / `data-change` 数据属性。不削弱 CSP，按钮在打包版恢复正常。
- 身份/信任库目录缺失不再是问题：core 的 `save_to_secure_path` / `save_to_path` 已自带 `create_dir_all`，首次「初始化身份」会自动创建应用数据目录。
- 验收：六个前端 JS `node --check` 通过；`cargo tauri build` 出包成功；已重装运行，并由用户在本机点击复验——「初始化身份 / 加载状态 / 开始监听」等按钮均可正常点击，os error 3 不再出现。

- 可复现的 Android `.so` 构建：补齐 NDK 28.2 + Rust 安卓 target + cargo-ndk，`core-rs` 可交叉编译 arm64-v8a / x86_64 并直接产出到 jniLibs；旧接口保持不变，桌面端/Tauri 零影响。
- debug APK 编译通过（gradle assembleDebug，含新 `.so`）。
- 模拟器端到端验收通过（`LinkHub_API_34` + adb 端口转发，Windows CLI 发送方）：Windows→Android 认证加密文件传输完成，Android 收到文件 SHA-256 一致，且**接收回调触发**——历史出现 `direction=received` 记录、通知发出（new JNI 路径 `get_java_vm`/`find_class`/`new_global_ref`/`onFileReceived` 全部验证）。

## 已完成修复

- 修复 JNI listener 和 desktop listener 预绑定后又让 core 重复绑定同一地址的问题。
- core 新增 `run_authenticated_listener_on`，允许上层传入已经绑定好的 `TcpListener`。
- Android 关闭系统备份，身份和信任设备数据迁移到加密 SharedPreferences。
- Android 前台服务会轮询 native listener 状态；后台线程退出或失败时会更新服务页状态并停止 mDNS 广播。
- Android 服务页中文文案已修正，监听状态、接收目录、本机地址和错误信息可直接用于模拟器/真机测试。
- Android 设备页和发送页中文文案已修正，可信设备扫描、地址刷新、文本发送和文件发送状态可读。
- Android NSD 发现已按 Android 14+ 接入 `registerServiceInfoCallback` 和 `hostAddresses`，旧系统保留兼容解析路径。
- 清理误提交的 Android/Gradle/Tauri 构建产物跟踪，并补充 `.gitignore`。
- README 和项目交接入口改为当前文档，不再依赖旧版专用提示词。
