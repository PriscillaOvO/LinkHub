## 本轮新增：2026-06-18 · 跨网络传输 M2-step1（信令服务器，新 crate）

- **背景**：选型已拍板 **webrtc-rs**（见下一节 spike），连带定了「信令服务器必须自建」。本步落地 M2 第一块：一个能本地跑、可验收的薄信令服务器，**先不接 WebRTC**（那是 M3）。
- **已完成（新增独立 crate [`signaling-server/`](../../signaling-server/)，tokio + tokio-tungstenite，72 依赖）**：
  - **Ed25519 登录鉴权**：服务器先发 `Challenge{nonce}`，设备回 `Auth{device_id, public_key_hex, signature_hex}`，`verify_strict` 校验。签名串域分隔为 `linkhub-signaling-auth-v1\0{nonce}`，与现有 p2p 握手签名（带双方 device_id）隔离，杜绝跨协议重放。

  - **隐私边界符合设计 §7**：服务器无任何密钥、看不到明文，最多知道元数据（谁在线/谁想连谁/何时）。
- **验收**：crate 内集成测试 `tests/forward.rs`——起服务器 + 两个 ws 客户端各自鉴权、A 经服务器转发 SIGNALING 给 B 并断言 B 收到（带 A 的已证明公钥），外加离线报错 / ping-pong / 坏签名拒绝；**7 单测 + 4 集成测试全绿**，`cargo fmt --check` / `cargo clippy --all-targets -D warnings` 干净。

## 本轮新增（2026-06-17 · 修复 Android 监听服务停止后无法重启 + release 版身份创建失败）

- 背景：真机测试中暴露两个 Android 端问题。
- 问题一 · 监听重启会卡住/重绑失败：`jni_bridge` 的 `startListener` 只把 `LISTENER_RUNNING` 置真后 `spawn` 工作线程，`stopListener` 只把标志置假，**不等待工作线程真正退出**。于是「停止 → 立即启动监听」时，旧线程可能仍持有已绑定的 socket，新线程 `TcpListener::bind` 报「address already in use」；并且旧线程晚退出时会把刚启动的新监听标志清零，导致状态显示错乱。
  - 修复：引入单一收口 `stop_and_join_listener()`——置 `LISTENER_RUNNING=false` 后 `join` 工作线程，确保 socket 已释放；`startListener` 启动前先调用它（处理上一线程尚在拆除的竞态），并把 `JoinHandle` 存入 `LISTENER_HANDLE`；用单调代次 `LISTENER_EPOCH` 做「代际保护」，工作线程退出时只有仍是当前代才清 `LISTENER_RUNNING`，被取代的旧线程晚退出不会再误关新监听；`stopListener` 改为调用同一收口，阻塞到线程退出后再返回。

## 本轮新增（2026-06-17 · 修复打包版按钮全部失效 / os error 3）

- 现象：安装版里「初始化身份」「开始监听」等按钮点了没反应，切到「设备」页加载状态报 `os error 3`（系统找不到路径）。
- 根因：所有交互按钮用的是行内 `onclick="..."` 事件属性。Tauri 打包构建会向 CSP 的 `script-src` 自动注入 nonce，按 CSP 规范，一旦存在 nonce，`'unsafe-inline'` 会被浏览器忽略，于是**行内事件处理器全部不触发**。标签页切换用的是 `addEventListener`（仍生效），所以切到「设备」页会自动触发 `renderDevicesTab()` → 因尚未初始化身份而报 os error 3，造成「按钮没反应、但加载状态有报错」的现象。
- 修复：在 `app.js` 增加全局事件委托（监听 document 的 click/change，按元素的 `data-act` / `data-change` + `data-a0..a3` 调用对应全局函数）；把六个前端文件里所有行内 `onclick` / `onchange` 改为 `data-act` / `data-change` 数据属性。不削弱 CSP，按钮在打包版恢复正常。
- 身份/信任库目录缺失不再是问题：core 的 `save_to_secure_path` / `save_to_path` 已自带 `create_dir_all`，首次「初始化身份」会自动创建应用数据目录。
- 验收：六个前端 JS `node --check` 通过；`cargo tauri build` 出包成功；已重装运行，并由用户在本机点击复验——「初始化身份 / 加载状态 / 开始监听」等按钮均可正常点击，os error 3 不再出现。
