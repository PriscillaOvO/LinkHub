# LinkHub 环境部署（从零拉取到跑通）

> 目标：一台**全新机器** + `git clone` 这个仓库后，照本文从上往下做，即可构建并验证每个组件。
> 开发主力机是 Windows，命令以 PowerShell 为主；跨平台差异在对应小节标注。
> 逐轮进展与当前能力见 [项目状态.md](项目状态.md)，整体架构见 [技术架构.md](技术架构.md)。

仓库组成（构建相关）：

```text
core-rs/            Rust core + CLI + JNI(.so) + iOS(.a)，所有平台共用的核心
signaling-server/   跨网络信令服务器（独立 Rust crate，可选，仅跨网传输用）
android/            Android App（Kotlin/Compose + Gradle）
desktop/            Tauri 2 桌面端（Rust + 纯静态 HTML/JS/CSS 前端）
ios/                iOS 脚手架（须 macOS 构建）
spike/              一次性交叉编译探雷，非主线，无需构建
```

---

## 0. 最短路径（只想先跑通 core + 桌面）

```powershell
# 1) 装 Rust（见 §1.1）+ Windows 上的 VS C++ Build Tools（见 §1.2）
# 2) clone 后：
cd C:\Dev\VSCode\LinkHub\core-rs
cargo test                       # 核心单测全绿即核心可用
cargo install tauri-cli --version "^2"
cd ..\desktop\src-tauri
cargo tauri dev                  # 起桌面端（首次编译较久）
```

Android / iOS / 跨网络是额外链路，按需看下面对应小节。

## 1. 必装工具链

| 工具 | 版本（基线） | 用途 | 安装 |
|---|---|---|---|
| Rust + Cargo | stable（≥1.83） | core / 桌面 / 信令 / CLI / 交叉编译 | https://rustup.rs |
| VS Build Tools（C++） | 2022，含「使用 C++ 的桌面开发」 | Windows 上 Rust/native 链接 | VS Installer |
| JDK | 17（Temurin/Adoptium） | Android Gradle 构建 | https://adoptium.net |
| Android SDK | 见 §1.4 组件清单 | Android App + NDK 交叉编译 | Android Studio 或 cmdline-tools |
| cargo-ndk | 4.1.x | 把 core 编成 Android `.so` | `cargo install cargo-ndk` |
| Tauri CLI | v2（`^2`） | 桌面端 dev/build/打包 | `cargo install tauri-cli --version "^2"` |
| WebView2 运行时 | 随 Win11 预装 | 桌面端 WebView | Win11 自带；旧系统装 Evergreen |
| Node.js（可选） | LTS | 仅用于前端 JS 的 `node --check` 静态检查 | 桌面构建本身**不需要** Node |

> iOS 需额外工具，且**只能在 macOS 上**，见 §6。
> 可选 feature（`webrtc` / `tor`）默认不编译，无需额外系统依赖（`tor` 的 `rusqlite` 用 `bundled` 自带 SQLite 源码），见 §7。

### 1.1 Rust 与交叉编译 target

```powershell
# 装好 rustup 后：
rustup target add aarch64-linux-android x86_64-linux-android   # Android .so
# iOS（仅 macOS 上需要）：
# rustup target add aarch64-apple-ios aarch64-apple-ios-sim x86_64-apple-ios
cargo install cargo-ndk
cargo install tauri-cli --version "^2"
```

仓库未固定 `rust-toolchain.toml`，用当前 stable 即可。

### 1.2 Windows C++ Build Tools

Rust 在 Windows 用 MSVC 链接器。装 **Visual Studio 2022 Build Tools** 并勾选「使用 C++ 的桌面开发」工作负载（提供 `link.exe` + Windows SDK）。

### 1.3 JDK 17

Android Gradle 需要 JDK 17（不是更高版本）。装 Temurin 17，设 `JAVA_HOME`（见 §2）。

### 1.4 Android SDK 组件

用 Android Studio 的 SDK Manager，或 `sdkmanager`（cmdline-tools）装齐：

```text
platform-tools
cmdline-tools;latest
platforms;android-34
build-tools;34.0.0
ndk;28.2.13676358          # 交叉编译 liblinkhub_core.so，须与构建脚本默认版本一致
system-images;android-34;google_apis;x86_64
emulator
```

创建一个 AVD（本仓库验证用名 `LinkHub_API_34`，API 34 / x86_64）：

```powershell
sdkmanager "system-images;android-34;google_apis;x86_64"
avdmanager create avd -n LinkHub_API_34 -k "system-images;android-34;google_apis;x86_64"
```

Android 构建基线（`android/app/build.gradle.kts` / 项目级）：AGP **8.13.2**、Kotlin **1.9.20**、Gradle wrapper **8.13**、`compileSdk 34` / `minSdk 26` / `targetSdk 34`、Java 17、Compose 编译器扩展 1.5.5。Gradle 由 `gradlew.bat` 自动下载，无需手装。

## 2. 环境变量（Windows，用户级）

```powershell
$sdk = "$env:LOCALAPPDATA\Android\Sdk"          # 默认安装位置；按实际改
$jdk = "C:\Program Files\Eclipse Adoptium\jdk-17.x.x-hotspot"  # 按实际改

setx ANDROID_HOME      $sdk
setx ANDROID_SDK_ROOT  $sdk
setx JAVA_HOME         $jdk
# ANDROID_NDK_HOME 可不设，构建脚本会按默认 NDK 版本推导；要手动设：
setx ANDROID_NDK_HOME  "$sdk\ndk\28.2.13676358"
```

把这些加进用户 `Path`：

```text
%ANDROID_HOME%\platform-tools
%ANDROID_HOME%\emulator
%ANDROID_HOME%\cmdline-tools\latest\bin
%JAVA_HOME%\bin
```

当前 PowerShell 会话临时注入（不想重开终端时）：

```powershell
$sdk = "$env:LOCALAPPDATA\Android\Sdk"
$jdk = "C:\Program Files\Eclipse Adoptium\jdk-17.x.x-hotspot"
$env:ANDROID_HOME = $sdk; $env:ANDROID_SDK_ROOT = $sdk; $env:JAVA_HOME = $jdk
$env:Path = "$jdk\bin;$sdk\platform-tools;$sdk\emulator;$sdk\cmdline-tools\latest\bin;$env:Path"
```

## 3. clone 后需要重新生成 / 确认的本地文件

这些被 `.gitignore` 排除（机器相关或构建产物），**新 clone 后不存在或需重建**：

| 文件 / 目录 | 怎么来 | 说明 |
|---|---|---|
| `android/local.properties` | 见下方模板 | 指向本机 SDK，**机器相关，必须本地生成** |
| `android/app/src/main/jniLibs/**/liblinkhub_core.so` | `scripts\build-android-so.ps1` | Android 运行所需 native 库，改了 `core-rs` 也要重跑 |
| `*/target/`、`android/app/build/`、`.gradle/` | 各构建命令自动生成 | 无需手动处理 |
| `desktop/src-tauri/gen/schemas/` | Tauri 构建自动生成 | 无需手动处理 |

`android/local.properties` 模板（按本机 SDK 路径改）：

```properties
sdk.dir=C:\\Users\\<用户名>\\AppData\\Local\\Android\\Sdk
```

> **签名密钥已随仓库提交**（私有仓库专门保留）：`android/app/linkhub-release.jks` + `android/keystore.properties`，clone 后即可直接出 release 包，无需重建。详见 §5 与 [项目状态.md](项目状态.md) 顶部说明。

## 4. 各组件构建与验证

### 4.1 core-rs（核心，必跑）

```powershell
cd C:\Dev\VSCode\LinkHub\core-rs
cargo fmt --all -- --check
cargo test                       # 默认矩阵（lib + CLI + e2e），基线全绿
cargo clippy --all-targets -- -D warnings
```

### 4.2 core-rs → Android `.so`

```powershell
cd C:\Dev\VSCode\LinkHub
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\build-android-so.ps1
# 等价手动命令：
#   $env:ANDROID_NDK_HOME = "$env:LOCALAPPDATA\Android\Sdk\ndk\28.2.13676358"
#   cd core-rs
#   cargo ndk -t arm64-v8a -t x86_64 -o ..\android\app\src\main\jniLibs build --release --lib
# 仅交叉编译校验（不产物）：
#   cargo ndk -t arm64-v8a -t x86_64 check --lib
```

### 4.3 signaling-server（可选，仅跨网络传输）

```powershell
cd C:\Dev\VSCode\LinkHub\signaling-server
cargo test
cargo run                        # 默认监听 ws://127.0.0.1:9000（看 src/main.rs）
```

### 4.4 desktop（Tauri 2）

前端是 `desktop/src` 下的纯静态 HTML/JS/CSS，**无 npm 步骤**。

```powershell
cd C:\Dev\VSCode\LinkHub\desktop\src-tauri
cargo tauri dev                  # 开发运行
cargo tauri build                # 打安装包（NSIS/MSI，WiX/NSIS 首次自动下载）
# 或用脚本：powershell -File ..\..\scripts\build-desktop-installer.ps1
cargo test                       # 桌面 smoke 测试
```

### 4.5 android

先确保 §4.2 的 `.so` 已生成、`local.properties` 已就位，再：

```powershell
cd C:\Dev\VSCode\LinkHub\android
.\gradlew.bat :app:assembleDebug                 # Debug APK
.\gradlew.bat :app:assembleRelease               # Release APK（用已提交的签名密钥）
# 或一键 release：powershell -File ..\scripts\build-android-release.ps1
```

模拟器烟测：

```powershell
$sdk = "$env:LOCALAPPDATA\Android\Sdk"
Start-Process "$sdk\emulator\emulator.exe" -ArgumentList @('-avd','LinkHub_API_34','-no-window','-no-snapshot','-gpu','swiftshader_indirect','-no-audio') -WindowStyle Hidden
adb wait-for-device
adb install -r .\app\build\outputs\apk\debug\app-debug.apk
adb shell monkey -p com.linkhub.app -c android.intent.category.LAUNCHER 1
adb shell pidof com.linkhub.app
adb emu kill
```

Android Studio 也可直接打开 `android/` 目录（用系统 `JAVA_HOME`）。双机/真机联调见 [真机测试指南.md](真机测试指南.md) 与 `scripts/two-device-app-test.md`。

## 5. Android 发布签名

签名密钥库与口令**已随本（私有）仓库提交**，clone 后即可复用同一签名出包：

- `android/app/linkhub-release.jks`（dev/test keystore，V2 签名）
- `android/keystore.properties`（store/key 口令，当前 `linkhub-dev-2026`）

> ⚠️ 这是 dev/test 密钥。**若日后要上架应用商店或把仓库转公开，必须轮换这把 key 并从 git 历史中清除**（口令是明文）。Android 要求同一签名才能给已安装 App 推更新，故重装机器时务必保留此仓库副本即可，无需新建 key。

## 7. 可选 feature 构建（默认关闭）

默认构建**不含** WebRTC 跨网络与 Tor onion，以保持精简的 `.so` 和测试矩阵。按需开启：

```powershell
# WebRTC 跨网络传输（core / 桌面 / CLI）：
cargo test --features webrtc                                  # core，含 WebRTC/TURN e2e
cargo ndk -P 24 -t arm64-v8a -t x86_64 build --release --lib --features webrtc   # 带 webrtc 的 .so（minSdk 24）
# 桌面：cargo tauri build 时给 src-tauri 加 --features webrtc

# Tor onion 传输（Arti，约 +200 依赖 crate）：
cargo build --features tor
cargo ndk --features tor -t arm64-v8a -t x86_64 check --lib
# CLI 子命令：listen-tor / connect-tor（须 --features tor）
```

各 feature 的设计与门控见 [设计-跨网络传输-webrtc.md](设计-跨网络传输-webrtc.md)、[设计-tor-onion-传输.md](设计-tor-onion-传输.md)。
